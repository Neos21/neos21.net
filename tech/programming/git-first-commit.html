Git は最初に空コミットする

リポジトリ作成直後、ブランチ作成直後は、ファイルに何の変更も入れずに、空コミットを入れる運用が良い。

$ git commit --allow-empty で、何の変更もない状態でコミットが打てる。

$ git commit --allow-empty -m 'Init'

コレを定型文にしよう。

最初のコミットを Rebase したくなった時にやりづらくなるのを避ける狙いと、ブランチ作成地点が後で探しやすくなるため、オススメである。
